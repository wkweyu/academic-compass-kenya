# Income Generating Activities (IGA) Module -- AI Implementation Work Plan

## Overview

This document defines the implementation plan for building an **Income
Generating Activities (IGA) module** inside a multi-tenant School
ERP system.\
The module manages income-generating school projects such as dairy
farming, poultry farming, horticulture, fish farming, and workshops.

The module must support: - production tracking - inventory management -
sales tracking - expense recording and approval - budgeting - accounting
integration - procurement integration - profitability reporting

All tables must include `school_id` to enforce tenant isolation.

------------------------------------------------------------------------

# Phase 2 --- Core Models

All tables must include:

    school_id
    created_at
    updated_at

## Activity Model

Represents a revenue-generating school project.

Fields:

    id
    school_id
    name
    description
    manager_id
    start_date
    status
    income_account_id
    expense_account_id
    inventory_account_id

Examples:

    Dairy Farming
    Poultry Unit
    Vegetable Garden
    Fish Farm

------------------------------------------------------------------------

## Product Model

Represents produce generated by activities.

Fields:

    id
    school_id
    name
    unit_of_measure
    sale_price
    inventory_account_id
    income_account_id

Examples:

    Milk
    Eggs
    Chicken
    Tomatoes

# Phase 3 --- Production Management

Model:

    ProductionRecord

Fields:

    id
    school_id
    activity_id
    product_id
    quantity
    unit
    production_date
    recorded_by

Example:

    Activity: Dairy
    Product: Milk
    Quantity: 120 Litres
    Date: 2026-05-01

-   creates inventory movement

# Phase 4 --- Inventory System

## InventoryStock

Tracks available stock.

Fields:

    id
    school_id
    product_id
    quantity_available
    unit
    last_updated

------------------------------------------------------------------------

## InventoryMovement

Tracks every inventory change.

Fields:

    id
    school_id
    product_id
    activity_id
    movement_type
    quantity
    reference
    date

Movement types:

    production
    sale
    internal_use
    spoilage
    adjustment

Example flow:

    Production +120L
    Sale -90L
    Kitchen use -10L
    Spoilage -5L

------------------------------------------------------------------------

# Phase 8 --- Expense Approval Workflow

Workflow:

    Staff records expense
    → Accountant reviews
    → Admin approves
    → Expense posted to accounting

Service functions:

    create_expense()
    approve_expense()
    reject_expense()

Only approved expenses post to accounting.

------------------------------------------------------------------------

# Phase 9 --- Accounting Integration

## Expense Entry

    Debit   Activity Expense Account
    Credit  Cash / Payables

Function:

    post_activity_expense()

------------------------------------------------------------------------

## Produce Sale Entry

    Debit   Cash / Bank
    Credit  Activity Income Account

Function:

    post_produce_sale()

------------------------------------------------------------------------

## Spoilage Entry

    Debit   Inventory Loss
    Credit  Produce Inventory

Function:

    post_inventory_loss()

------------------------------------------------------------------------

## Internal Consumption Entry

Example: milk used in school kitchen.

    Debit   Kitchen Supplies
    Credit  Produce Inventory

------------------------------------------------------------------------
